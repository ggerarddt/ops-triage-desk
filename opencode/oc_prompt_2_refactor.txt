Refactor the Operational Incident Triage Desk into a more production-shaped structure without making it large.

Context:
Evolve from a useful prototype into a maintainable service shape while preserving the same user-visible behavior, login flow, seeded SQLite users, role-based navigation, and example auto-fill buttons.

Refactoring goals:
- Separate HTTP/UI code from business logic.
- Separate authentication/session helpers from incident triage logic.
- Move SQLite schema creation and seed data into focused database code.
- Move severity, routing, summary, and integration logic into focused modules.
- Move configurable mappings into a configuration module.
- Add simple domain models.
- Add a JSON API endpoint for downstream/internal automation.
- Add structured logging.

Required structure:
- app.py
  - Flask bootstrap only: create the app and register routes.
- router.py
  - Route registration and request/response handling.
  - Keep parsing and presentation helpers small and readable.
- core/
  - models.py for Incident, TriageResult, and User dataclasses.
  - config.py for severity descriptions, team keywords, default team, seed incidents, and seed users.
  - database.py for SQLite connection, schema initialization, and seed loading.
  - summary_service.py for the handoff summary builder.
- services/
  - auth_service.py for user lookup and password verification.
  - severity_service.py for classify_severity() and get_severity_description().
  - routing_service.py for route_team() and escalation_recommendation().
  - triage_service.py for orchestration of the full triage pipeline.
- integrations/
  - mock_integrator.py for build_mock_payload().
- templates/
  - login.html
  - index.html
  - result.html
  - audit.html
  - admin.html
- static/app.js if needed for example button behavior.
- requirements.txt
- README.md

Functional requirements:
- Preserve the existing login, logout, triage form, result view, audit view, and supervisor-only admin navigation.
- Preserve the seeded SQLite users:
  - operator1 / ChangeMe123! / operator
  - operator2 / ChangeMe123! / operator
  - supervisor1 / ChangeMe123! / supervisor
- Add POST /api/triage that accepts JSON from an authenticated user and returns JSON with:
  - original normalized input
  - severity
  - suggested team
  - escalation recommendation
  - structured summary
  - mock payload
- Keep the same severity rules and routing teams.
- Keep the three demo seed incidents and example auto-fill buttons.
- The mock integration adapter should only build a payload. It must not perform a real HTTP call.

Engineering requirements:
- Use dataclasses for domain objects.
- Use dependency direction that keeps services independent from Flask.
- Keep database access small and explicit.
- Avoid global business logic in app.py.
- Keep route handlers thin enough to explain.
- Log triage completion with incident severity and team.
- Do not log raw sensitive incident descriptions if avoidable.
- Preserve simple local execution with python app.py.

Regression requirement:
- While refactoring, protect the example auto-fill behavior. Do not leave the buttons as inert submit buttons. If JavaScript is used, load it from the template correctly and test the DOM ids/names used by the script.
